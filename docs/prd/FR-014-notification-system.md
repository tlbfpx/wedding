# FR-014: 消息通知系统

> 优先级：P1
> 所属模块：消息通知
> 关联用户故事：US-003, US-004, US-005
> 状态：待开发

---

## 功能描述

建立站内消息通知系统，当关键业务事件发生时自动通知相关人员。支持审批待办提醒、排期变更提醒、客户跟进提醒三类核心通知场景。

通知采用轮询模式（前端定时拉取），不引入 WebSocket，降低实现复杂度。

---

## 输入

### 创建通知（系统内部调用）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_id | int | 是 | 接收通知的用户ID |
| title | string | 是 | 通知标题 |
| content | string | 是 | 通知内容 |
| type | string | 是 | 通知类型：approval / schedule / follow_up / system |
| related_id | int | 否 | 关联业务ID（审批ID/排期ID/客户ID） |
| related_type | string | 否 | 关联业务类型：approval / event / customer |

### 查询通知

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| is_read | bool | 否 | 是否已读筛选 |
| type | string | 否 | 通知类型筛选 |
| page | int | 否 | 页码 |
| page_size | int | 否 | 每页条数，默认20 |

---

## 业务规则

1. **审批通知**：审批创建时通知审批人，审批处理时通知申请人
2. **排期通知**：排期创建/变更时通知相关策划师和销售
3. **跟进提醒**：客户 next_follow_at 到期时通知负责销售（每日定时检查）
4. 通知保留 90 天，超期自动清理
5. 已读通知支持批量标记
6. 前端导航栏显示未读计数气泡，30 秒轮询刷新
7. 通知支持"点击跳转到关联业务"（前端路由跳转）

---

## 异常场景

| 场景 | 触发条件 | 处理方式 |
|------|----------|----------|
| 用户不存在 | 通知目标用户已停用 | 仍创建通知，用户重新启用后可见 |
| 批量标记已读 | 传入不属于自己的通知ID | 忽略非本人的通知ID |

## 验收标准

| 编号 | 验收条件 | 验证方式 |
|------|----------|----------|
| AC-001 | 提交审批后审批人收到站内通知 | 创建审批，检查审批人通知列表 |
| AC-002 | 审批通过/驳回后申请人收到通知 | 审批决定，检查申请人通知 |
| AC-003 | 创建/修改排期后相关策划师收到通知 | 创建排期，检查通知 |
| AC-004 | 导航栏未读计数实时显示 | 产生新通知后检查气泡数字 |
| AC-005 | 点击通知跳转到关联业务页面 | 点击审批通知跳转审批详情 |
| AC-006 | 批量标记已读功能正常 | 选择多条通知标记已读 |
