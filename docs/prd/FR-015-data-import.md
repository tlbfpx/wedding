# FR-015: 数据导入

> 优先级：P1
> 所属模块：数据管理
> 关联用户故事：US-006, US-007
> 状态：待开发

---

## 功能描述

支持通过上传 Excel/CSV 文件批量导入客户和供应商基础数据，降低手工录入成本。提供数据校验、错误反馈和导入结果汇总。

---

## 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| import_type | string | 是 | 导入类型：customer / supplier |
| file | file | 是 | Excel(.xlsx) 或 CSV(.csv) 文件 |

### 客户导入模板列

| 列名 | 必填 | 说明 |
|------|------|------|
| 姓名 | 是 | name |
| 手机号 | 是 | phone，唯一校验 |
| 性别 | 否 | gender：男/女 |
| 来源 | 否 | source |
| 预算范围 | 否 | budget_range |
| 婚期 | 否 | wedding_date，YYYY-MM-DD |
| 备注 | 否 | note |

### 供应商导入模板列

| 列名 | 必填 | 说明 |
|------|------|------|
| 名称 | 是 | name |
| 类型 | 是 | type：四大金刚/婚车/场地/花艺/摄影/主持/其他 |
| 联系人 | 否 | contact |
| 联系电话 | 否 | phone |
| 地址 | 否 | address |
| 合作状态 | 否 | cooperation_status：合作中/已暂停/已拉黑，默认合作中 |
| 备注 | 否 | note |

---

## 输出

### 导入结果

| 字段 | 类型 | 说明 |
|------|------|------|
| total | int | 文件中总行数（不含表头） |
| success | int | 成功导入行数 |
| failed | int | 失败行数 |
| errors | array | 错误明细列表 |

### 错误明细

| 字段 | 类型 | 说明 |
|------|------|------|
| row | int | 行号（从2开始，1为表头） |
| field | string | 出错字段 |
| message | string | 错误描述 |

---

## 业务规则

1. 文件大小限制 5MB
2. 单次导入上限 1000 行
3. 必须包含表头行，表头名称必须匹配模板
4. 手机号重复的行跳过并记录错误（不更新已有记录）
5. 枚举值不匹配的行报错（如性别不是"男/女"）
6. 提供模板下载接口，用户先下载模板再填写
7. 导入成功的客户默认状态为"潜在"，不指派销售
8. 导入成功的供应商默认评分为空

---

## 异常场景

| 场景 | 触发条件 | 处理方式 |
|------|----------|----------|
| 文件格式错误 | 非 xlsx/csv 文件 | 返回 400 提示文件格式不正确 |
| 文件过大 | 超过 5MB | 返回 400 提示文件过大 |
| 数据量过大 | 超过 1000 行 | 返回 400 提示行数超限 |
| 表头不匹配 | 列名与模板不一致 | 返回 400 提示缺少必填列 |
| 全部失败 | 所有行均有错误 | 返回导入结果（success=0, errors 列表完整） |

## 验收标准

| 编号 | 验收条件 | 验证方式 |
|------|----------|----------|
| AC-001 | 下载客户导入模板，格式正确 | 下载并检查 Excel 表头 |
| AC-002 | 上传正确的客户数据，导入成功 | 准备5条客户数据导入验证 |
| AC-003 | 手机号重复的行报错并跳过 | 导入含重复手机号的数据 |
| AC-004 | 上传错误的表头返回友好提示 | 上传列名错误的文件 |
| AC-005 | 供应商导入功能正常 | 准备供应商数据导入验证 |
| AC-006 | 部分成功时返回成功和失败明细 | 混合正确和错误数据导入 |
