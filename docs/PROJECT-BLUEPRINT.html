婚礼策划管理系统 — 项目蓝图

📅 生成时间：2026-06-04 🔍 分析深度：standard 🎯 分析范围：全项目 💻 Python FastAPI + Vue 3 + MySQL

🚀 快速入门（新人必读 Top 10）

这 10 条信息能让新人在 30 分钟内建立项目基本认知

  1. 一句话定位：面向婚礼策划公司的全流程业务管理系统，覆盖客户跟进、档期管理、订单合同、财务核算、供应商协作等核心业务
  2. 技术栈：后端 Python 3.13 + FastAPI 0.115 + SQLAlchemy 2.0 async + MySQL 8 + Redis 7；前端 Vue 3.5 + TypeScript + Naive UI + Pinia
  3. 项目结构：Monorepo 双服务拓扑 — wedding-backend/（后端）+ wedding-frontend/（前端），后端采用 DDD 四层架构
  4. 如何启动：docker compose up -d 启动全部服务（MySQL + Redis + Backend + Frontend），或分别 uvicorn app.main:app 和 npm run dev
  5. 核心业务：CRM 客户跟进 → 档期/资源调度 → 订单合同签订 → 财务应收/退款/发票 → 供应商协作，全程审批流驱动
  6. 数据模型：25 张 MySQL 表，核心聚合根为 Customer、Order、Event，支撑 7 种角色、4 个团队的业务协作
  7. 外部依赖：MySQL 8（主存储）、Redis 7（缓存/JWT 黑名单/限流/事件总线）、Sentry（错误监控）、Prometheus（指标采集）
  8. 部署方式：Docker Compose 编排 4 个服务（frontend/nginx、backend/uvicorn、mysql、redis），支持健康检查和资源限制
  9. 关键配置：DATABASE_URL、REDIS_URL、JWT_SECRET、JWT_ACCESS_TOKEN_EXPIRE_MINUTES（120min）、BCRYPT_ROUNDS（12）
  10. 常见坑点：JWT 双 Token 机制（access + refresh）、异步 SQLAlchemy 需注意 session 生命周期、RBAC 权限粒度为 module+action+scope 三维
